## 🟠 높음 (이번 주 안에)

### ✅ T-003 · `apiRequest` 에러 응답 처리 부재 (완료)
- **위치**: `src/main.js` `apiRequest()` 함수
- **문제**: 서버가 4xx/5xx를 돌려줘도 상위에서 각자 처리해야 하고, 처리 안 하면 조용히 실패함. 에러 핸들링 패턴이 호출처마다 달라서 일부는 오류를 완전히 무시함
- **해결**: `apiRequest`에서 4xx/5xx 시 표준 에러 throw + 전역 에러 토스트 표시
- **상태**: 완료 (2026-02-23)
- **난이도**: 보통

### ✅ T-004 · 세션 만료 없음 (절대 만료 시간 없이 슬라이딩 연장만 있음) (완료)

- **해결**: `createdAt` 기준 절대 만료(예: 30일) 추가

### ✅ T-005 · 버전 충돌(409) 시 로컬 변경사항 강제 덮어쓰기 (완료)
- **위치**: `src/main.js` `applyServerStateSnapshot()`
- **문제**: 서버와 버전 충돌 발생 시 서버 상태로 완전히 덮어써서 사용자가 작성 중이던 내용이 사라짐
- **해결**: 충돌 시 로컬-서버 병합(merge) 로직 구현 또는 사용자에게 선택 UI 제공
- **상태**: 완료 (2026-02-23, 사용자 선택 UI + 충돌 백업 저장)
- **난이도**: 어려움

### ✅ T-006 · 협업 API에 Rate Limit 없음 (완료)
- **위치**: `server/modules/collab/router.js`
- **문제**: 고유ID 변경(`PUT /api/collab/public-id`), 초대 발송 등 협업 엔드포인트에 Rate Limit이 없어 스팸 가능
- **해결**: 협업 라우터에 Rate Limit 미들웨어 추가(예: 분당 10회)
- **상태**: 완료 (2026-02-23)
- **난이도**: 쉬움

### ✅ T-007 · 보안 이벤트 로그 디스크 가득 차면 조용히 유실 (완료)
- **위치**: `server.js` 로그 기록 함수
- **문제**: `appendFileSync` 실패를 silent catch로 무시. 디스크 풀이나 권한 문제 시 보안 로그가 유실됨
- **해결**: 로그 실패 시 `console.error` 출력 + 로그 파일 크기 제한(rotate) 추가
- **상태**: 완료 (2026-02-23)
- **난이도**: 쉬움

## 🏗️ 장기 로드맵 (규모가 커지면)

| 항목 | 설명 |
|------|------|
| SQLite → PostgreSQL | 동시 사용자 증가 시 전환 필요 |
| Redis 세션 저장소 | 서버 재시작 시 세션 유지 |
| 실시간 협업 (WebSocket) | 폴링 방식을 WebSocket으로 전환 |
| 모듈형 UI 아키텍처 | `src/main.js` 단일 파일을 컴포넌트로 분리 |
| 타입 안전성 (TypeScript/tRPC) | 프론트-서버 타입 공유 |
